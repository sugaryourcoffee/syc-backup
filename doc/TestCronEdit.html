    class TestCronEdit

    Test for CronEdit will add commands to the users crontab. After each test the test entries are removed. So the crontab should not be changed after a test. In case of changes to CronEdit that will lead to a system exit it might leave the crontab with test data. In that case you have to remove the test data manually. But subsequent successfull runs should remove all test entries.

    crontab_count(command) click to toggle source

    Determines the count of the provided command in the crontab.

    # File test/test_cron_edit.rb, line 16
def crontab_count(command)
  content = %xcrontab -l`
  count = 0
  content.split(%r\n/).each do |c|
    count += 1 if c == command
  end
  count
end
